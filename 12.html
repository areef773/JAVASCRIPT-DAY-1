1. var Function-scoped Can be re-declared and updated Gets hoisted (initialized as undefined) Older way of declaring variables 2. let Block-scoped ({ }) Can be updated, but not re-declared in the same scope Hoisted but not initialized (temporal dead zone) 3. const Block-scoped Cannot be updated or re-declared Must be initialized at declaration Used for fixed values In short: var → flexible but outdated let → use when value may change const → use when value should not change
